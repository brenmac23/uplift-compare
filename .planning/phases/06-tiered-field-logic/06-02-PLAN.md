**Field generation logic (in this exact order to ensure deterministic PRNG consumption):**

1. **hasStudioLease (B1):** `rand() < 0.10` (unchanged — test requires 3-5 out of 50)

2. **regionalPercent (B5):** `rand() < 0.5 ? 25 + Math.round(rand() * 50) : Math.round(rand() * 24)` — always consumes 2 `rand()` calls. If the first roll is >= 0.5 (low regional), still consume the second roll but use the result for the low range: `const r1 = rand(); const r2 = rand(); if (r1 < 0.5) { 25 + Math.round(r2 * 50) } else { Math.round(r2 * 24) }`.

3. **B6/B7 bimodal post-production (DIST-01):** Import `POST_PRODUCTION_CONFIG`. Always consume exactly 4 `rand()` calls:
   - `const clusterRoll = rand()` — determines high/low cluster (< 0.5 = high)
   - `const splitRollPicture = rand()` — 10% chance picture diverges from cluster
   - `const splitRollSound = rand()` — 10% chance sound diverges from cluster
   - `const valueRoll = rand()` — used for both picture and sound value within their cluster (or use a 5th roll for sound value if you want independent values within the cluster — but to keep PRNG stable, let both use the same roll for the value offset; the split already introduces variation)

   Actually, use 5 calls for cleaner separation:
   - `clusterRoll = rand()` — high/low
   - `pictureSplitRoll = rand()` — picture may diverge
   - `soundSplitRoll = rand()` — sound may diverge
   - `pictureValueRoll = rand()` — value within cluster for picture
   - `soundValueRoll = rand()` — value within cluster for sound

   ```
   const isHighCluster = clusterRoll < POST_PRODUCTION_CONFIG.highLowSplitProb;
   const pictureHigh = (pictureSplitRoll < POST_PRODUCTION_CONFIG.splitProb) ? !isHighCluster : isHighCluster;
   const soundHigh = (soundSplitRoll < POST_PRODUCTION_CONFIG.splitProb) ? !isHighCluster : isHighCluster;
   const picturePostPercent = pictureHigh
     ? POST_PRODUCTION_CONFIG.highRange.min + Math.round(pictureValueRoll * (highRange.max - highRange.min))
     : POST_PRODUCTION_CONFIG.lowRange.min + Math.round(pictureValueRoll * (lowRange.max - lowRange.min));
   // same for sound using soundValueRoll
   ```

4. **B8 vfxPercent (DIST-02):** `Math.round(rand() * 100)` — fully independent, uniform 0-100. NOT coupled to B6/B7.

5. **B9 conceptPhysicalPercent (DIST-02):** `Math.round(rand() * 100)` — fully independent, uniform 0-100.

6. **castPercent (C1, DIST-05):** Budget-inverse. Use a probability that decreases with budget tier:
   - small: 0.80 chance of >= 80%, mid: 0.70, large: 0.55, tentpole: 0.40
   - `const castProb = { small: 0.80, mid: 0.70, large: 0.55, tentpole: 0.40 }[tierConfig.tier]`
   - `const r = rand(); castPercent = r < castProb ? 80 + Math.round(rand() * 20) : Math.round(rand() * 79)`
   - Always consume 2 `rand()` calls (pre-read both, use second only if high).

7. **crewPercent (C2, DIST-04):** Import `B4_C2_COVARIANCE`. Check `tier1.shootingNZPercent`:
   - If >= `highShootingThreshold` (75): use `highShootingCrewPassProb` (0.88)
   - Else: use `lowShootingCrewPassProb` (0.35)
   - `const r1 = rand(); const r2 = rand(); crewPercent = r1 < passProb ? 80 + Math.round(r2 * 20) : Math.round(r2 * 79)`
   - Always consume 2 `rand()` calls.

8. **btlKeyCount and btlAdditionalCount (C5/C6, DIST-03):** Import `BTL_CONFIG`.
   - Determine `isHighPost = picturePostPercent >= 75` (from step 3 above)
   - `c5Max = isHighPost ? BTL_CONFIG.c5MaxWithPost : BTL_CONFIG.c5MaxWithoutPost`
   - Apply budget bias: `effectiveMax = Math.max(1, Math.round(c5Max * BTL_CONFIG.budgetC5Bias[tierConfig.tier]))`
   - `const c5Roll = rand(); const c5 = Math.floor(c5Roll * (effectiveMax + 1))`
   - If c5 === 0: `btlAdditionalCount = 0; rand()` (consume one roll and discard for sequence stability)
   - Else: `const c6Roll = rand(); btlAdditionalCount = c5 + Math.floor(c6Roll * (BTL_CONFIG.c6Max - c5 + 1))`
   - Always consume 2 `rand()` calls.

9. **supportingCastCount (C8, DIST-05):** Import `BUDGET_INVERSE_SUPPORTING_CAST_WEIGHTS`. Use `weightedSelect(rand(), weights[tierConfig.tier])` — 1 `rand()` call.

**Total PRNG consumption per project in Tier 2:** 1 (B1) + 2 (B5) + 5 (B6/B7) + 1 (B8) + 1 (B9) + 2 (C1) + 2 (C2) + 2 (C5/C6) + 1 (C8) = 17 `rand()` calls. This count must be deterministic regardless of field values.
