  <name>Task 1: Install Tailwind v4 + shadcn/ui and configure project</name>
  <files>vite.config.ts, tsconfig.app.json, src/index.css, src/lib/utils.ts, components.json</files>
  <action>
1. Add path aliases to tsconfig.app.json: `"baseUrl": "."` and `"paths": { "@/*": ["./src/*"] }` inside compilerOptions.

2. Update vite.config.ts: import `path` from `'path'` and `tailwindcss` from `'@tailwindcss/vite'`. Add `tailwindcss()` to plugins array. Add `resolve: { alias: { '@': path.resolve(__dirname, './src') } }`.

3. Install Tailwind v4: `npm install tailwindcss @tailwindcss/vite`

4. Run shadcn init: `npx shadcn@latest init --defaults` (this creates components.json, src/lib/utils.ts with cn() helper). If it prompts, select: style=new-york, base-color=neutral, css-variables=yes.

5. Replace the ENTIRE contents of src/index.css with:
```css
@import "tailwindcss";

@theme {
  --color-pass: oklch(0.6 0.15 240);
  --color-fail: oklch(0.65 0.18 55);
  --color-pass-bg: oklch(0.95 0.03 240);
  --color-fail-bg: oklch(0.95 0.04 55);
}
```
Do NOT keep any of the old CSS variables or styles. Tailwind + shadcn handle everything now. The old `:root` variables, body styles, h1/h2/code rules are all replaced by Tailwind utilities and shadcn theme.

6. Add all required shadcn components in one command:
```bash
npx shadcn@latest add button dialog select tooltip collapsible badge card table
```

7. Verify the build compiles: `npm run build`

IMPORTANT: Use Tailwind v4 syntax (`@import "tailwindcss"`) NOT v3 directives (`@tailwind base`). See 03-RESEARCH.md Pitfall 1.
IMPORTANT: Configure path aliases BEFORE running shadcn init. See 03-RESEARCH.md Pitfall 2.
  </action>
